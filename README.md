## Introduction

Do you have a handful of directories you work in every day — your dev root, 
several project roots, a log directory? Do you find yourself setting up shell
aliases that have to be updated regularly, or using tab-completion to reach
a deep directory, or opening multiple terminals, tabs, or views to avoid having
to navigate back and forth?

Park Directories solves this with simple bookmarks. Assign a short name to any
directory, then jump to it instantly from anywhere:

```
pd -a dev                         # bookmark the current directory as "dev"
pd -a log /var/log/myapp          # bookmark any path
pd dev                            # jump there instantly
pd dev/src/components             # jump to a subdirectory of "dev"
pd -l                             # list all bookmarks
pd -d log                         # remove a bookmark
```

### Relative Path Navigation
You don't need to bookmark every directory you visit. Park a project root and
navigate to any subdirectory relative to it — `pd proj/src`, `pd proj/tests`,
`pd proj/docs` — without individual bookmarks for each one. Tab completion
extends into subdirectories, so exploration is fast.

### Scripting
Park Directories is designed with scripting in mind. Use `pd -x name` to expand
a bookmark to its full path without navigating — handy for building up shell
commands.
```nushell
## nushell
cp $"(pd -x proj/target/release/bin.exe)" ~/local/bin
```

### Technology Overview
The core is a compiled Rust binary that handles all logic. A thin shell
function — generated by the binary itself — performs the `cd` call, since no
external process can change your shell's working directory directly. All shells
share a single bookmark file, so bookmarks you add in one shell are immediately
available in all others.

## Installation

### Get the binary

**Option 1 — Download a pre-built binary (recommended)**

Download the latest release from the
[Releases page](https://github.com/nb-twy/ParkDirectories/releases/latest):

| Platform | File |
|---|---|
| Windows x64 | `pd-x86_64-pc-windows-msvc.exe` |
| Linux x64 | `pd-x86_64-unknown-linux-musl` |
| Linux ARM64 | `pd-aarch64-unknown-linux-musl` |

Rename the file to `pd` (or `pd.exe` on Windows) and copy it to a directory in
your PATH:

- **Windows:** `C:\Users\you\bin\pd.exe` (where `C:\Users\you\bin` is in `$env:PATH`)
- **Linux:** `~/.local/bin/pd` (where `~/.local/bin` is in `$PATH`)

**Option 2 — Install with cargo**

Requires [Rust](https://rustup.rs) (rustup is recommended).

```
cargo install park-directories
```

The binary is installed as `pd` in `~/.cargo/bin/`, which rustup adds to your
PATH automatically. To update to the latest version, re-run the same command.

**Option 3 — Build from source**

```
git clone https://github.com/nb-twy/ParkDirectories.git
cd ParkDirectories
cargo build --release
```

Copy `target/release/pd` (or `target\release\pd.exe` on Windows) to a directory
in your PATH.

### Nushell — Windows and Linux

**Step 1 — [Get the binary](#get-the-binary)** (see above)

**Step 2 — Add to your nushell config**

Add these two lines to `~/.config/nushell/config.nu`:

```nushell
^pd init nu | save -f ~/.config/nushell/pd.nu
source ~/.config/nushell/pd.nu
```

The first line regenerates the integration script from the binary on every
shell startup, so the integration stays in sync automatically whenever you
update `pd`. You can store `pd.nu` anywhere you like — just use the same path
in both lines.

> **Note:** `~/.config/nushell/` does not exist by default. Before adding
> these lines, either create the directory (`mkdir ~/.config/nushell`) or
> choose a directory that already exists and adjust the path accordingly.
> `config.nu` itself is typically found at the path reported by
> `$nu.config-path`.

**Step 3 — Restart your terminal**

**Updating**

Update the binary (if installed via cargo: `cargo install park-directories`),
then restart your terminal. The integration script regenerates automatically
on startup — no additional steps required.

---

### Bash — Linux

**Step 1 — [Get the binary](#get-the-binary)** (see above)

**Step 2 — Add shell integration to your bashrc**

```bash
echo 'eval "$(pd init bash)"' >> ~/.bashrc
source ~/.bashrc
```

**Updating**

Update the binary (if installed via cargo: `cargo install park-directories`),
then restart your terminal or run:

```bash
eval "$(pd init bash)"
```

### PowerShell — Windows

**Step 1 — [Get the binary](#get-the-binary)** (see above)

**Step 2 — Add the PowerShell integration to your profile**

Add the following line to your PowerShell profile (`$PROFILE`):

```powershell
& pd init pwsh | Out-String | Invoke-Expression
```

To find your profile path: `echo $PROFILE`. Create the file if it does not
exist.

**Step 3 — Restart your terminal** (or run `. $PROFILE`)

**Tab completion style**

By default, PowerShell cycles through completions one at a time inline. PSReadLine
(included with PowerShell 5.1+) lets you change this. The `MenuComplete` mode opens
a navigable popup list — the closest experience to nushell's completion menu.

| Mode | Behavior |
|---|---|
| `TabCompleteNext` (default) | Cycles through matches inline, one at a time |
| `Complete` | Completes immediately if only one match; lists all after double `Tab` if ambiguous |
| `MenuComplete` | Opens a navigable popup list — select with arrow keys and `Enter` |

To always use the popup list, add this to your `$PROFILE`:

```powershell
Set-PSReadLineKeyHandler -Key Tab -Function MenuComplete
```

Below is the author's preference.

```powershell
Set-PSReadLineKeyHandler -Key Tab        -Function Complete
Set-PSReadLineKeyHandler -Key Shift+Tab  -Function TabCompletePrevious
Set-PSReadLineKeyHandler -Key Ctrl+Spacebar -Function MenuComplete
```

**Updating**

Update the binary (if installed via cargo: `cargo install park-directories`),
then restart your terminal or run `. $PROFILE` to re-evaluate the integration.

## How to Use

### Commands

| Command | Short flag | What it does |
|---|---|---|
| `pd name` | | Navigate to a bookmarked directory |
| `pd name/relative/path` | | Navigate to a subdirectory of a bookmark |
| `pd add name` | `pd -a name` | Bookmark the current directory as `name` |
| `pd add name /some/path` | `pd -a name /path` | Bookmark a specific path |
| `pd del name` | `pd -d name` | Remove a bookmark |
| `pd list` | `pd -l` | List all bookmarks |
| `pd clear` | `pd -c` | Clear all bookmarks (prompts for confirmation) |
| `pd expand name` | `pd -x name` | Print the resolved path without navigating |
| `pd export file` | `pd -e file` | Export bookmarks to a file |
| `pd import file` | `pd -i file` | Import bookmarks from a file |
| `pd import --append file` | `pd -i --append file` | Merge imported bookmarks with existing |

All commands accept both short flags (`-a`, `-d`, `-l`, etc.) and long flags
(`--add`, `--del`, `--list`, etc.).

Bookmark names may not start with `-` or contain `/`.

---

### Navigating relative paths

Park a project root and navigate to any subdirectory without individual
bookmarks:

```
pd -a proj /home/user/dev/my-project

pd proj/src
pd proj/tests
pd proj/docs/getting-started
```

---

### Tab completion
| What you type | What completes |
|---|---|
| `pd <Tab>` | Bookmark names |
| `pd name/<Tab>` | Subdirectories of the bookmarked path |
| `pd -a name <Tab>` | Filesystem directories |
| `pd -d <Tab>` | Bookmark names |
| `pd -x <Tab>`, `pd -x name/<Tab>` | Bookmark names / subdirectories (same as navigation) |
| `pd -<Tab>`, `pd --<Tab>` | Available flags |

Tab completion extends into subdirectories: `pd proj/<Tab>` lists the
top-level subdirectories of the bookmarked path.  `pd proj/src/<Tab>`
lists the contents of `src`.

---

### Bookmark file

Bookmarks are stored in `~/.pd-data` (created automatically on first use). The
format is plain text — one entry per line — and is human-readable and
hand-editable:

```
work   C:\Users\you\projects\work
dots   C:\Users\you\.config
logs   C:\ProgramData\MyApp\logs
```

All shells share this file by default. To use a different file, set the
`PD_DATA_FILE` environment variable or pass `--data-file <path>` to any
command.

---

### Scripting

`pd expand name` (or `pd -x name`) prints the resolved path without navigating,
making it easy to use bookmarks in shell commands:

```nushell
## nushell
cp -r $"(pd -x src)/assets" $"(pd -x build)/assets"
```
---

### Export and import

Export your bookmarks for backup or to transfer to another machine:

```
pd -e my-bookmarks.txt
```

Import with `--append` to merge with your existing bookmarks:

```
pd -i --append my-bookmarks.txt
```

Import without `--append` for an interactive overwrite, which lets you choose
to back up the current file, overwrite directly, or abort:

```
pd -i my-bookmarks.txt
```
